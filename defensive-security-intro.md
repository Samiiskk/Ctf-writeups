## What is Defensive Security?
Also called the **Blue Team**.
Defenders protect and monitor systems instead of attacking them.

**Core cycle:**
Monitor → Detect → Investigate → Contain → Recover → Improve

## Main Responsibilities
- Monitor networks and systems
- Investigate suspicious activity
- Respond to security incidents
- Prevent future attacks through security improvements

## Key Tools
| Tool | Purpose |
|------|---------|
| SIEM | Collects logs, generates alerts from one dashboard |
| Firewall | Controls and blocks network traffic |
| IDS | Detects suspicious activity, generates alerts |
| IPS | Detects AND automatically blocks malicious traffic |

## Defence in Depth
Multiple layers of security — if one fails, another stops the attacker.
Examples: employee training, firewalls, IDS/IPS, monitoring

## Incident Response Lifecycle
1. Preparation
2. Detection & Analysis
3. Containment
4. Eradication
5. Recovery
6. Post-Incident Review

## Key Takeaway
IDS detects. IPS detects AND blocks.
First action after detecting an attack — Investigation & Analysis.
Immediate action during an attack — Containment.
